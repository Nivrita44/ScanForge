SCANFORGE - BANGLA OCR & DOCUMENT INTELLIGENCE SYSTEM
======================================================
Development Documentation: Final OCR & Parsing Model

1. CORE ARCHITECTURE
-------------------
The system uses a two-tier extraction strategy to handle both native (text-based) and scanned (image-based) PDFs:
- Tier 1: Native Extraction (pdf-parse v1.1.1). Fast, handles text layers.
- Tier 2: OCR Fallback (pdf-img-convert + Tesseract.js). Used when text layers are absent or sparse.

2. BANGLA NORMALIZATION ENGINE (The "Parser Core")
-------------------------------------------------
Most Bangla PDFs store text in "Visual Order" (vowel signs before consonants). The ScanForge Normalizer performs a multi-pass transformation to restore Unicode Logical Order:

A. Vowel Shifting:
   - Moves pre-base markers (ি, ে, ৈ) after their base consonants (e.g., েক -> কে).
   - Recomposes split vowels (ে + কা -> কো).

B. Conjunct Restoration (Jugmo Borno):
   - Restores missing hasantas (connecting marks) in common clusters.
   - Ganj (গঞ্জ): Maps extracted 'গঞ' -> 'গঞ্জ'.
   - Master (মাস্টার): Maps common misreadings like 'মাষার' and 'মাষাব' -> 'মাস্টার'.
   - Reph Restoration: Specifically maps 'ণর' -> 'র্ণ' for name and address accuracy.

3. NID DUAL-SIDE EXTRACTION
---------------------------
The system now supports multi-file uploads (Front + Back) for NID cards:
- Logic: Backend merges OCR text from both images into a single processing block.
- Field Separation: Surgically separates 'Blood Group' (O+/A-/etc.) from 'Place of Birth' (জন্মস্থান) even if they appear on the same line.
- Manual Refinement: Every field in the resulting form is editable for 100% data accuracy.

4. INSTANT CAMERA MODULE
-------------------------
- Implementation: Uses `navigator.mediaDevices.getUserMedia` for real-time video preview.
- Frame Capture: Uses a hidden <canvas> to grab high-resolution JPEG snapshots (0.95 quality) from the video stream.
- Scanning UI: Professional overlay with CSS-animated 'laser' scanning line and alignment frame.

5. MOBILE-FIRST ARCHITECTURE
-----------------------------
- UI Framework: Responsive Vanilla CSS with dynamic grid-based layouts.
- Breakpoints: Optimized for Desktop, Tablet (768px), and mobile (640px).
- Navigation: Dashboard-first 'Portal' layout (সেবাসমূহ) with persistent headers and welcome bars.

6. DATA EXPORT & COMPATIBILITY
-----------------------------
- Excel Formula Trick: Long numeric Voter IDs are exported as `="<ID>"` to prevent Excel's scientific notation conversion.
- UTF-8 BOM: Injects `\uFEFF` into the CSV file to ensure Excel renders Bangla characters correctly.

7. REGEX FIELD MAPPING (REFINED)
---------------------------------
- Patterns for Serial Numbers, Names, Voter IDs, and Addresses.
- Optimized NID patterns: 
  - ID Number: Support for 10, 13, and 17 digit variants.
  - Blood Group: Isolated regex `\b(A|B|AB|O)[\s]*([\+\-])\b` to avoid keyword pollution.
  - Jonmosthan: Specifically strips trailing 'Print/মুদ্রণ' artifacts from place of birth.
